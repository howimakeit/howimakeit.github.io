<h1 style='text-align: center;'>一、基础核心语法</h1>
<h3>1. 切换数据库（USE）</h3>
<ul>
<li> 语法：
<pre><code class="language-sql">USE 数据库名;</code></pre>
</li>
<li> 大白话：告诉 MySQL “我要操作哪个数据库”，必须加分号。</li>
<li> 示例：
<pre><code class="language-sql">USE sql_store;</code></pre>
</li>
<li> 避坑：忘写分号→MySQL 一直等待输入，报错1046。</li>
</ul>

<h3>2. 注释（写备注，不执行）</h3>
<ul>
<li> 单行注释：-- 注释内容（必须加空格）</li>
<li> 多行注释：/* 注释内容 */</li>
<li> 示例：-- 查询所有商品</li>
</ul>

<h3>3. 基础查询（SELECT）</h3>
<ul>
<li> 查所有列：
<pre><code class="language-sql">SELECT * FROM 表名;</code></pre>
</li>
<li> 查指定列：
<pre><code class="language-sql">SELECT 列1,列2 FROM 表名;</code></pre>
</li>
<li> 自定义列名：
<pre><code class="language-sql">SELECT 列名 AS '新列名' FROM 表名;</code></pre>
</li>
<li> 去重查询：
<pre><code class="language-sql">SELECT DISTINCT 列名 FROM 表名;</code></pre>
</li>
<li> 示例：
<pre><code class="language-sql">SELECT DISTINCT country FROM Websites;</code></pre>
</li>
</ul>

<h3>4. 条件筛选（WHERE）</h3>
<ul>
<li> 作用：像if判断，只保留符合条件的数据。</li>
<li> 比较符：= &gt; &lt; &gt;= &lt;= !=</li>
<li> 逻辑符：AND（优先级&gt;OR）、OR、NOT</li>

<li> 多条件：
<ul>
<li> IN：WHERE 列 IN (值1,值2) → 等价多个OR</li>
<li> BETWEEN...AND：WHERE 列 BETWEEN 最小值 AND 最大值（闭区间）</li>
</ul>
</li>

<li> 模糊查询：
<ul>
<li> LIKE：%= 任意字符，_= 单个字符（全字段匹配才返回）</li>
<li>示例：
<pre><code class="language-sql">WHERE first_name LIKE 'S%'</code></pre>
（S 开头）
</li>
<li> REGEXP：正则匹配（部分匹配就返回） ^开头、$结尾、|或者、[]任意字符</li>
</ul>
</li>

<li> 空值判断：WHERE 列 IS NULL（不能用 = NULL）</li>
</ul>

<h3>5. 排序（ORDER BY）</h3>
<ul>
<li> 语法：
<pre><code class="language-sql">SELECT * FROM 表名 ORDER BY 列名 [ASC/DESC];</code></pre>
</li>
<li> ASC：升序（默认，可省略）；DESC：降序</li>
<li> 多列排序：
<pre><code class="language-sql">ORDER BY 列1 DESC,列2 ASC</code></pre>
</li>
<li> 避坑：必须放在查询最后一句，否则报错。</li>
</ul>

<h3>6. 分页（LIMIT）</h3>
<ul>
<li> 语法：
<pre><code class="language-sql">LIMIT 跳过行数, 取几条;</code></pre>
（起始行是 0）
</li>
<li> 示例：
<pre><code class="language-sql">LIMIT 6,2</code></pre>
→跳过前 6 条，取后面 2 条
</li>
<li> 避坑：别记反 “先取后跳”。</li>
</ul>

<h1 style='text-align: center;'>二、表连接</h1>

<p>核心：表与表靠相同列（如 ID）关联。</p>

<h3>1. 内连接（INNER JOIN）</h3>
<ul>
<li>语法：
<pre><code class="language-sql">SELECT 列 FROM 表1 JOIN 表2 ON 表1.关联列=表2.关联列;</code></pre>
</li>

<li>作用：只返回两张表都匹配的数据，不匹配直接丢弃。</li>

<li>示例：
<pre><code class="language-sql">SELECT o.order_id, c.
first_name 
FROM
 orders o 
JOIN customers c ON o.customer_id = c.customer_id;</code></pre>
</li>

<li>别名：表名太长可起别名（o=orders），起别名后全程用别名。</li>
</ul>

<h3>2. 外连接（LEFT/RIGHT JOIN）</h3>
<ul>
<li>LEFT JOIN（左连接，最常用）：保留左表所有数据，右表无匹配→显示NULL</li>

<li>RIGHT JOIN（右连接）：保留右表所有数据，左表无匹配→显示NULL</li>

<li>MySQL 无全外连接：用LEFT JOIN + UNION + RIGHT JOIN实现</li>

<li>示例（查所有顾客，含无订单顾客）：￼
<pre><code class="language-sql">SELECT c.first_name, o.
order_id 
FROM
 customers c 
LEFT JOIN orders o ON c.customer_id = o.customer_id;</code></pre>
</li>
</ul>

<h3>3. 其他连接</h3>
<ul>
<li>自连接：同一张表起别名连接（员工 - 经理）</li>

<li>跨库连接：库名.表名（如sql_inventory.products）</li>

<li>复合连接：JOIN ON 条件1 AND 条件2（多列关联）</li>

<li>交叉连接：CROSS JOIN→所有行两两组合</li>

<li>USING：简化连接JOIN 表 USING(关联列)（列名必须相同）</li>
</ul>

<h1 style='text-align: center;'>三、数据增删改（DML，修改数据库数据）</h1>

<h3>1. 插入数据（INSERT）</h3>
<ul>
<li>单行插入：
<pre><code class="language-sql">INSERT INTO 表(列1,列2) VALUES(值1,值2);</code></pre>
</li>

<li>多行插入：
<pre><code class="language-sql">VALUES(值1),(值2),(值3);</code></pre>
</li>

<li>忽略重复：
<pre><code class="language-sql">INSERT IGNORE INTO 表 VALUES(值);</code></pre>
（主键重复不报错）
</li>

<li>关联插入：LAST_INSERT_ID()→获取最新自增 ID</li>

<li>示例：
<pre><code class="language-sql">INSERT INTO customers(first_name,last_name) VALUES('John','Smith');</code></pre>
</li>
</ul>

<h3>2. 更新数据（UPDATE高危操作）</h3>
<ul>
<li>语法：
<pre><code class="language-sql">UPDATE 表 SET 列1=值1,列2=值2 WHERE 条件;</code></pre>
</li>

<li>避坑：必须加 WHERE，不加会修改全表所有数据，无法恢复！</li>

<li>示例：
<pre><code class="language-sql">UPDATE customers SET points=1000 WHERE customer_id=1;</code></pre>
</li>
</ul>

<h3>3. 删除数据（DELETE 高危操作）</h3>
<ul>
<li>删指定行：
<pre><code class="language-sql">DELETE FROM 表 WHERE 条件;</code></pre>
</li>

<li>清空全表：
<pre><code class="language-sql">TRUNCATE TABLE 表;</code></pre>
（比 DELETE 快 10 倍，不记录日志）
</li>

<li>避坑：TRUNCATE无法回滚，慎用！</li>
</ul>

<h3>4. 复制表</h3>
<ul>
<li>语法：
<pre><code class="language-sql">CREATE TABLE 新表 AS SELECT * FROM 原表;</code></pre>
</li>

<li>缺点：丢失主键、自增、约束。</li>
</ul>

<h1 style='text-align: center;'>四、聚合与分组</h1>

<h3>1. 聚合函数（返回 1 个结果）</h3>
<ul>
<li>MAX()  最大值</li>
<li>
<pre><code class="language-sql">SELECT MAX(points) FROM customers;</code></pre>
</li>

<li>MIN()  最小值</li>
<li>
<pre><code class="language-sql">SELECT MIN(points) FROM customers;</code></pre>
</li>

<li>AVG()  平均值</li>
<li>
<pre><code class="language-sql">SELECT AVG(points) FROM customers;</code></pre>
</li>

<li>SUM()  求和</li>
<li>
<pre><code class="language-sql">SELECT SUM(points) FROM customers;</code></pre>
</li>

<li>COUNT()  计数</li>
<li>COUNT (*) 算所有行；COUNT (列) 不算 NULL</li>

<li> 避坑：聚合函数不能放在 WHERE 里！里面得子查询</li>
</ul>

<h3>2. 分组（GROUP BY）</h3>
<ul>
<li>语法：
<pre><code class="language-sql">SELECT 分组列,聚合函数 FROM 表 GROUP BY 分组列;</code></pre>
</li>

<li>硬性规则：SELECT 列只能是分组列或聚合函数，否则报错。</li>

<li>示例：按国家统计顾客数
<pre><code class="language-sql">SELECT country, COUNT(*) FROM customers GROUP BY country;</code></pre>
</li>
</ul>

<h3>3. 分组后筛选（HAVING）</h3>
<ul>
<li>语法：
<pre><code class="language-sql">GROUP BY 分组列 HAVING 条件;</code></pre>
</li>

<li>区别：
<ul>
<li>WHERE：分组前过滤（无聚合）</li>
<li>HAVING：分组后过滤（可聚合）</li>
</ul>
</li>

<li>示例：
<pre><code class="language-sql">HAVING COUNT(*)>5</code></pre>
</li>
</ul>

<h3>4. 分组汇总（WITH ROLLUP）</h3>
<ul>
<li>语法：
<pre><code class="language-sql">GROUP BY 列 WITH ROLLUP;</code></pre>
</li>

<li>作用：自动添加总计行。</li>
</ul>

<h1 style='text-align: center;'>五、子查询（嵌套查询）</h1>

<h3>1. 定义</h3>
<p>一个查询里套另一个查询，内层结果给外层用。</p>

<h3>2. 常用位置</h3>
<ul>
<li>WHERE 子查询：条件里嵌套</li>

<li>示例：查高于平均工资的员工</li>
<li>
<pre><code class="language-sql">SELECT * FROM employees WHERE salary>(SELECT AVG(salary) FROM employees);</code></pre>
</li>

<li>FROM 子查询：结果当表用（必须起别名）</li>
<li>SELECT 子查询：计算列值</li>
</ul>

<h3>3. 关键运算符</h3>
<ul>
<li>ALL：满足子查询所有结果</li>
<li>ANY：满足任一结果（=ANY等价IN）</li>
<li>EXISTS：判断子查询是否有结果（大数据量比IN快）</li>
</ul>

<h1 style='text-align: center;'>六、内置函数</h1>

<h3>1. 数值函数</h3>
<ul>
<li>ROUND(数值,位数)：四舍五入</li>
<li>TRUNCATE(数值,位数)：截断</li>
<li>CEILING()：向上取整；FLOOR()：向下取整</li>
</ul>

<h3>2. 字符串函数</h3>
<ul>
<li>UPPER()/LOWER()：大小写转换</li>
<li>CONCAT()：拼接；CONCAT_WS()：带分隔符拼接（忽略 NULL）</li>
<li>SUBSTRING(列,起始,长度)：截取字符串</li>
<li>TRIM()：去除前后空格</li>
</ul>

<h3>3. 日期函数</h3>
<ul>
<li>NOW()：当前日期 + 时间；CURDATE()：当前日期</li>
<li>YEAR()/MONTH()/DAY()：提取年月日</li>
<li>DATEDIFF(日期1,日期2)：计算日期差</li>
</ul>

<h3>4. 空值处理</h3>
<ul>
<li>IFNULL(列,默认值)：NULL 替换为默认值</li>
<li>COALESCE(列1,列2,默认值)：返回第一个非 NULL 值</li>
</ul>

<h3>5. 流程函数</h3>
<ul>
<li>IF(条件,真,假)：单条件判断</li>
<li>CASE WHEN 条件1 THEN 结果1 ... ELSE 结果 END：多条件判断</li>
</ul>

<h1 style='text-align: center;'>七、存储过程、事务、触发器</h1>

<h3>（一）存储过程（PROCEDURE）→ 批量执行 SQL</h3>

<h3>1. 定义</h3>
<p>把多条 SQL 封装成一个函数，需要时直接调用，不用重复写代码。</p>

<h3>2. 完整语法</h3>
<pre><code class="language-sql">
DELIMITER $$
CREATE PROCEDURE 过程名(参数)
BEGIN
  SELECT * FROM customers;
END
$$
DELIMITER ;

CALL 过程名();
</code></pre>

<h3>3. 参数类型</h3>
<ul>
<li>IN：输入参数（给过程传值）</li>
<li>OUT：输出参数（过程返回结果）</li>
</ul>

<h3>4. 变量声明</h3>
<p>DECLARE 变量名 类型 DEFAULT 默认值;（仅存储过程内可用）</p>

<h3>5. 删除存储过程</h3>
<p>DROP PROCEDURE IF EXISTS 过程名;</p>

<h3>（二）事务（TRANSACTION）→ 保证数据安全</h3>

<h3>1. 定义</h3>
<p>一组操作要么全成功，要么全失败，不会出现 “一半成功一半失败”（如银行转账）。</p>

<h3>2. 核心特性（ACID）</h3>
<ul>
<li>原子性：不可分割</li>
<li>一致性：数据始终合法</li>
<li>隔离性：并发不干扰</li>
<li>持久性：提交后永久生效</li>
</ul>

<h3>3. 完整语法</h3>
<pre><code class="language-sql">
START TRANSACTION;

UPDATE users SET money=money-100 WHERE id=1;
UPDATE users SET money=money+100 WHERE id=2;

COMMIT;

-- ROLLBACK;
</code></pre>

<h3>4. 并发问题</h3>
<ul>
<li>丢失更新、脏读、不可重复读、幻读</li>
</ul>

<h3>5. 隔离级别</h3>
<p>READ UNCOMMITTED → READ COMMITTED → REPEATABLE READ（默认）→ SERIALIZABLE</p>

<h3>6. 死锁</h3>
<ul>
<li>原因：两个事务反向锁定同一批数据，互相等待</li>
<li>解决：统一数据更新顺序、缩短事务时长</li>
</ul>

<h3>（三）触发器（TRIGGER）→ 自动执行 SQL</h3>

<h3>1. 定义</h3>
<p>表发生增删改时，自动触发执行预设 SQL（如插入订单→自动更新库存）。</p>

<h3>2. 核心关键字</h3>
<ul>
<li>NEW：新增 / 修改后的新行数据</li>
<li>OLD：删除 / 修改前的旧行数据</li>
</ul>

<h3>3. 完整语法</h3>
<pre><code class="language-sql">
DELIMITER $$
CREATE TRIGGER 触发器名
[BEFORE/AFTER] [INSERT/UPDATE/DELETE] ON 表名 FOR EACH ROW
BEGIN
  UPDATE invoices 
  SET payment_total=payment_total+NEW.amount 
  WHERE invoice_id=NEW.invoice_id;
END
$$
DELIMITER ;
</code></pre>

<h3>4. 触发器类型</h3>
<ul>
<li>AFTER INSERT：插入后触发</li>
<li>BEFORE DELETE：删除前触发</li>
<li>AFTER UPDATE：修改后触发</li>
</ul>

<h3>5. 触发器审计</h3>
<p>作用：记录数据修改日志（用OLD记录修改前数据）</p>

<h3>6. 管理触发器</h3>
<ul>
<li>SHOW TRIGGERS;</li>
<li>DROP TRIGGER IF EXISTS 触发器名;</li>
</ul>

<h1 style='text-align: center;'>八、数据类型</h1>

<h3>1. 字符型</h3>
<ul>
<li>CHAR(n)：定长（如性别CHAR(1)）</li>
<li>VARCHAR(n)：变长（如姓名VARCHAR(50)）</li>
<li>TEXT：长文本（文章、备注）</li>
</ul>

<h3>2. 数值型</h3>
<ul>
<li>INT：整数（ID、积分）</li>
<li>DECIMAL(p,s)：定点数（金额，如DECIMAL(10,2)）</li>
<li>BOOL：布尔值（1 = 真，0 = 假）</li>
</ul>

<h3>3. 日期型</h3>
<ul>
<li>DATE：日期（2024-01-01）</li>
<li>DATETIME：日期 + 时间（2024-01-01 12:00:00）</li>
</ul>

<h3>4. 特殊类型</h3>
<ul>
<li>JSON：键值对数据</li>
<li>BLOB：二进制文件（图片、音频）</li>
</ul>

<h1 style='text-align: center;'>九、索引与约束（优化 + 规范数据）</h1>

<h3>1. 索引（加快查询）</h3>
<ul>
<li>普通索引：CREATE INDEX 索引名 ON 表(列);</li>
<li>唯一索引：CREATE UNIQUE INDEX 索引名 ON 表(列);</li>
<li>全文索引：CREATE FULLTEXT INDEX 索引名 ON 表(列);</li>
<li>避坑：索引太多会减慢增删改，别乱加。</li>
</ul>

<h3>2. 约束</h3>
<ul>
<li>主键：PRIMARY KEY（唯一、非空，一张表一个）</li>
<li>外键：FOREIGN KEY(列) REFERENCES 主表(列)</li>
<li>ON UPDATE CASCADE：主键更新，外键同步</li>
<li>ON DELETE NO ACTION：禁止删除主表数据</li>
</ul>

<h1 style='text-align: center;'>十、数据库设计</h1>

<h3>1. 三范式</h3>
<ul>
<li>1NF：列不可拆分</li>
<li>2NF：表单一目的，无重复数据</li>
<li>3NF：无传递依赖</li>
</ul>

<h3>2. 关系设计</h3>
<ul>
<li>多对多→加中间表拆成一对多</li>
</ul>

<h3>3. 工程化</h3>
<ul>
<li>正向工程：模型→建表</li>
<li>逆向工程：表→生成模型</li>
</ul>

<h1 style='text-align: center;'>十一、用户权限管理</h1>
<ul>
<li>创建用户：CREATE USER 用户名@IP IDENTIFIED BY '密码';</li>
<li>授权：GRANT 权限 ON 库.表 TO 用户;</li>
<li>撤权：REVOKE 权限 FROM 用户;</li>
<li>查看权限：SHOW GRANTS;</li>
</ul>

<h1 style='text-align: center;'>十二、SQL 执行顺序</h1>
<ul>
<li>写 SQL 顺序：SELECT → FROM → WHERE → GROUP BY → HAVING → ORDER BY → LIMIT</li>
<li>MySQL 执行顺序：FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT</li>
</ul>

<h1 style='text-align: center;'>十三、致命</h1>
<ol>
<li>语句忘写分号→执行失败</li>
<li>UPDATE/DELETE不加 WHERE→全表被改 / 删</li>
<li>空值用=NULL→永远查不到</li>
<li>GROUP BY后用非分组列→报错</li>
<li>多表连接未加表前缀→列名冲突</li>
<li>LIMIT起始行从 1 开始→数据取错</li>
<li>子查询在FROM里未起别名→报错</li>
<li>日期未加引号→报错</li>
<li>聚合函数放在WHERE里→报错</li>
<li>UNION两张表列数不一致→报错</li>
</ol>

<h1 style='text-align: center;'>十四、后面出题目咯</h1>
<ol>
<li>用户留存率：LEFT JOIN+日期计算+去重</li>
<li>分组最值：窗口函数RANK()/ 子查询关联</li>
<li>字符串拆分：SUBSTRING_INDEX+IF</li>
<li>中位数：PERCENTILE_CONT(0.5)+ 排序</li>
<li>连续登录：LAG()/LEAD()窗口函数</li>
</ol>
